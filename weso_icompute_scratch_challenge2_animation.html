The bouncing patrol

Create a program where a sprite patrols the screen, bouncing off the edges, while a second sprite chases it. When the chaser catches the patrol sprite, something happens!

1
Choose two sprites. The patrol sprite moves forward 5 steps and bounces off edges in a forever loop. It should also slowly rotate as it moves.
2
The chaser sprite always points toward the patrol sprite and moves forward 3 steps in a forever loop — it's a little slower so the patrol can escape!
3
When the chaser is touching the patrol sprite, it should stop both sprites, play a sound, and say "Got you!".
4
Add a backdrop and a variable on screen showing how many seconds have passed since the program started.
Key idea: The two sprites each need their own set of blocks — both starting with when 🏳 clicked. Use touching [sprite]? inside an if block to detect the catch.
Challenge + Above & beyond — targets creativity/efficiency points

Avoid the obstacles

Create a simple game where the player controls a sprite using the arrow keys to avoid randomly-moving obstacles. The game ends — and shows the player's score — when the player sprite touches an obstacle.

1
The player sprite moves using the arrow keys (up, down, left, right), each key moving the sprite 10 steps in that direction.
2
Create two obstacle sprites. Each one starts at a random position, moves in a random direction, and bounces off edges in a forever loop.
3
Use a score variable that increases by 1 every second the player survives. Show it on screen.
4
When the player touches either obstacle, stop all sprites, play a sound, and have the player sprite say "Game over! Score: [score]" using join.
5
Add a backdrop and make the obstacles change color continuously while they move — this earns creativity points.
Efficiency tip: Use a repeat until <touching player?> loop for each obstacle instead of a forever + if — it's cleaner and judges notice that kind of thing for the tiebreaker.

Scratch coordinate system: The stage is 480 wide × 360 tall. The center is (0, 0). Left edge is x = −240, right edge is x = 240. Top edge is y = 180, bottom edge is y = −180.
Two sprites, two scripts: Each sprite has its own script area. Click a sprite in the sprite list, then add blocks for that sprite only. When green flag is clicked, all sprites start their own scripts at the same time.

CategoryPointsWhat judges look for in animation tasks
Correct algorithm 10 pts Right constructs used — forever loops for continuous motion, if/touching for collision detection, point towards for chasing. Logic flows correctly.
Working program 10 pts Sprites actually move when green flag is clicked. Collisions trigger correctly. No frozen or runaway sprites.
Precision 5 pts Correct coordinates used, timing is right (wait blocks match the spec), messages spelled correctly, edge-bounce works smoothly.
Creativity & efficiency 5 pts Tiebreaker. Costume changes during movement, color effects, backdrops, sounds on collision, clean loop structure instead of repeated code.
Total Section C: 30 points  ·  Full test total: 50 points
Coach tip: For animation tasks, "working program" points are easy to lose if sprites get stuck at edges. Make sure students always pair move ( ) steps with if on edge, bounce inside their forever loop.
