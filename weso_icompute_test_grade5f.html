Programming Task — Dice Tournament
Scratch 3.29 offline

Build a 3-round dice tournament where you compete against the computer! Each round both you and the computer roll a die — whoever rolled higher wins that round. Most rounds won at the end wins the tournament. Your program must do all of the following:

1
Green flag script: Create variables player_score, computer_score, and round. Set all three to 0. Make player_score and computer_score visible on the stage. Switch to a tournament backdrop. Say "Get ready — 3 rounds!" for 2 seconds.
2
Add a repeat 3 times loop. At the top of each loop iteration:
  • Change round by 1
  • Create variables my_roll and comp_roll
  • Set my_roll to pick random 1 to 6
  • Set comp_roll to pick random 1 to 6
3
Inside the loop, use if / else (or nested if) to compare rolls:
  • If my_roll > comp_roll → change player_score by 1, say (join "Round " (join round ": You win!")) for 2 seconds, play a cheer sound
  • Else if my_roll = comp_roll → say "Tie round!" for 2 seconds
  • Else → change computer_score by 1, say "Computer wins this round!" for 2 seconds, play a different sound
4
After the loop (outside it), check the final result:
  • If player_score > computer_score → switch backdrop to a winner scene, say "You won the tournament!" for 3 seconds
  • Else → switch backdrop to a loser scene, say "Computer wins the tournament!" for 3 seconds
Scoring rubric — 30 pts total
Correct algorithm (player_score, computer_score, round all start at 0; repeat 3 loop present; my_roll and comp_roll each use pick random 1–6; round increments each iteration; if/else correctly awards scores; post-loop if/else checks final winner; join used in round result message)10 pts
Working program (3 rounds play out with visible score updates; player winning a round increments player_score; computer winning increments computer_score; after 3 rounds the correct winner message and backdrop appear; both sounds play in the correct situations)10 pts
Precision (player_score and computer_score visible throughout; round counter changes each iteration; join correctly builds the round result message; post-loop winner check is outside the repeat block, not inside; two different sounds used)5 pts
Creativity & efficiency — tiebreaker (at least 2 distinct backdrops; tournament intro backdrop different from winner/loser backdrops; code clean; extra polish such as a tie backdrop or score display labels)5 pts
